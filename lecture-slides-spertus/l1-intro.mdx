## Software Engineering is the Integral of Programming Over Time

<p style={{marginTop: '0.75em', fontSize: '0.95em'}}>
  This explains why we care about <em>changeability</em>, <em>readability</em>, and <em>testing</em>
</p>
<p style={{marginTop: '0.75em', fontSize: '0.9em'}}>
  Questions that seem "non-technical" become engineering concerns:
</p>
<ul style={{fontSize: '0.85em', marginTop: '0.25em'}}>
  <li>Who uses our software?</li>
  <li>Who maintains it?</li>
  <li>What assumptions do we bake in?</li>
</ul>
<p style={{marginTop: '0.5em', fontSize: '0.85em', color: '#666'}}>
  These costs compound over time
</p>

<aside class="notes">
**Titus Winters' framing** (former Google, now Adobe, co-author "Software Engineering at Google"):
- Code written once and thrown away ‚Üí can be sloppy
- Real software lives years/decades, maintained by changing teams

**Bad decisions compound:**
- Hard-to-read code slows every future change
- Untested code accumulates bugs
- Biased assumptions affect users for years

**Why principles over syntax:** Principles help software age well

‚Üí **Transition:** This brings us to software sustainability...

## Most Software Cost Is Maintenance, Not Initial Development

<p style={{fontSize: '1.1em', color: '#9370DB'}}>
  <strong>Sustainability</strong> ‚Äî the ability of software to continue providing value over time
</p>
<p style={{marginTop: '1em', fontSize: '0.95em'}}>
  Four dimensions we'll consider:
</p>
<div style={{display: 'grid', gridTemplateColumns: '1fr 1fr', gap: '0.75em', marginTop: '0.5em', fontSize: '0.85em'}}>
  <div><strong>Technical:</strong> Can we maintain it?</div>
  <div><strong>Economic:</strong> Can we afford it?</div>
  <div><strong>Social:</strong> Does it serve people well?</div>
  <div><strong>Environmental:</strong> What resources does it consume?</div>
</div>
<p style={{marginTop: '1em', fontSize: '0.9em', color: '#666'}}>

<aside class="notes">

**Reframes everything:** We're creating something others (including future selves) must understand, modify, operate

## JIT Combines Interpretation with Native Compilation

<p style={{fontSize: '1.1em'}}>
  <strong>Just-In-Time (JIT)</strong> compilation
</p>
<p style={{marginTop: '1em'}}>
  Dynamically compiles bytecode <em>as it executes</em>

<aside class="notes">
**JIT = Java's innovation:**
- JVM starts interpreting, monitors "hot" code (frequently run)

**Bonus:** JVM optimizes based on runtime behavior (ahead-of-time compilers can't!)
